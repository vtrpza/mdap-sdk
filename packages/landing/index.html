The Problem

Single LLM Calls Fail

Even the best models make mistakes. At scale, those mistakes compound into system-wide unreliability.

94%

Per-Call Accuracy

GPT-4 achieves ~94% accuracy on classification. Sounds impressive in isolation.

100x

Operations Scale

Run 100 operations at 94%. Probability of zero errors: just 0.2%.

6

Expected Failures

Six misrouted tickets. Six wrong classifications. Six unhappy customers.

The Mathematics of Failure

P(all correct) = 0.94100 = 0.2%

At 94% per-operation accuracy, running 100 independent operations gives you only a 0.2% chance of getting everything right. This is why agentic systems feel unreliable in production.

The Solution

Reliability Through Consensus

Like distributed systems achieve fault tolerance through redundancy, MDAP achieves reliability through intelligent voting.

First-to-Ahead-by-K Voting

1

Sample

Execute the same LLM call multiple times in parallel, collecting independent responses.

2

Vote

Count responses as they arrive. Continue until one option leads by k votes.

3

Consensus

Return the winning response with a confidence score derived from the vote margin.

Early termination = lower costs | Higher k = higher accuracy

Quick Start

Get Started

Add enterprise-grade reliability to your LLM applications in minutes.

1

Install

npm install @mdap/core
2

Wrap Your LLM Calls

import { reliable, RedFlag } from '@mdap/core';

// Create a reliable classifier
const classify = reliable<string>({
  vote: {
    k: 3,
    strategy: 'first-to-ahead-by-k',
    maxSamples: 30
  },
  redFlags: [
    RedFlag.emptyResponse(),
    RedFlag.tooLong(100),
  ]
})(async (input: string) => {
  return await llm.chat(`
    Classify this ticket: ${input}
    Categories: BILLING, TECHNICAL, SHIPPING, ACCOUNT
  `);
});

// Use it like any async function
const result = await classify("My payment failed");
console.log(result); // "BILLING" with 99.8% reliability

Design for Convergence

Voting requires responses that can converge. Use constrained categories, not open-ended prompts.

// Won't converge
"Identify bugs in this code"
// Will converge
"Classify: BUG | SMELL | NONE"
